# ゆとりBASE 報酬分配アプリ MVP

スマホでLK支払明細PDFを読み込み、ARB報酬の配分結果、入出金スケジュール、振込明細を作るテスト版です。

## できること

- LK明細PDFから文字を読取る。
- 顧客名、代理店コード、プラン、申込月、初期手数料、月額手数料を確認する。
- `YBY001` と `YBY002` から `YBY005` の報酬配分を計算する。
- 申込時の月額2か月分を、対象月ごとの支払予定に分ける。
- 初期費用は申込入金月の翌月末、月額は対象月末の支払いとして扱う。
- 今回の支払月に該当する振込額だけを振込明細に出す。
- 支払先の銀行名、支店名、口座番号、名義、インボイス番号を代表のスマホ内だけに保存する。
- 振込明細、報酬配分明細、支払通知書を印刷用ページとして出す。
- スマホの印刷/共有機能でPDF保存する。

## できないこと

- 画像PDF、スキャンPDFのOCR。
- 銀行振込の実行。
- ChatGPT APIや外部AIによる読取。
- `YBY006` 以降の2次代理店配分。
- 口座振替の実データとの自動照合。
- 複数端末間での支払先設定の自動同期。
- 税務上の正式判断。

## 代表向け操作手順

1. アプリURLをスマホで開く。
2. SafariまたはChromeのメニューからホーム画面に追加する。
3. 初回だけ `支払先設定` を開き、銀行名、支店名、口座番号、名義、インボイス番号を入力して端末に保存する。
4. `LK明細PDFを選択` を押す。
5. PDFを選ぶ。
6. `今回の支払月` を確認する。
7. 読取結果を確認する。
8. `配分を計算` を押す。
9. `振込明細を表示` を押す。
10. 印刷画面または共有画面からPDFとして保存する。
11. 保存した振込明細を見ながら、銀行アプリで振込する。
12. 支払通知PDFを必要な相手へLINEまたはメールで送る。

## 読取できない場合

画面に `このPDFは自動読取できません。オーナーへ送ってください。` と出たら、代表は手入力しません。

その場合は、LK明細PDFをオーナーへ送って、Codex代行処理で振込明細を作成します。

## テスト方法

1. `サンプル明細で試す` を押す。
2. `配分を計算` を押す。
3. 以下を確認する。
   - 5月申込の初期費用分は、6月末支払いになる。
   - `YBY001` 初期70,000円は、ゆとりBASE35,000円、ヨウスケさん35,000円、オーナー0円。
   - `YBY002` 初期70,000円は、ゆとりBASE21,000円、ヨウスケさん21,000円、オーナー28,000円。
   - 6月末の振込合計は、ゆとりBASE56,000円、ヨウスケさん56,000円、オーナー28,000円。
   - 7月末と8月末は、前受け月額分として各月3者それぞれ2,000円。
   - `YBY006` は要確認で止まる。

## 支払先設定

- アプリ本体やGitHubには、実際の口座番号や個人情報を入れません。
- 代表がスマホで入力した支払先情報は、その端末のブラウザ内に保存します。
- 保存される項目は、表示名、インボイス番号、銀行名、支店名、口座種別、口座番号、口座名義、備考です。
- スマホを変更した場合やブラウザデータを削除した場合は、再入力が必要です。
- 実運用時は、スマホの画面ロック、GitHubの2段階認証、銀行アプリの認証を必ず有効にします。

## 実運用前に確認するもの

- ゆとりBASEの正式名称、住所、インボイス番号。
- オーナーの請求書名義、住所、インボイス登録有無、振込先。
- ヨウスケさんの請求書名義、住所、インボイス登録有無、振込先。
- 支払日、締め日、振込手数料負担。
- 源泉徴収、消費税、請求書保存ルール。

## 技術メモ

- PDF読取はブラウザ内で行います。
- PDF本文や顧客情報は外部APIへ送信しません。
- 支払先設定は `localStorage` に保存します。サーバー保存やGitHub保存はしません。
- PDF読取ライブラリの読み込みにはインターネット接続が必要です。
- 出力PDFは、ブラウザの印刷/PDF保存機能で作成します。
